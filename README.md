# dotfiles

- `nvim/` - Neovim (LazyVim)
- `micro/` - Micro editor
- `shell/` - Zsh, environment variables
- `bash/` - Bash
- `tmux/` - Tmux
- `git/` - Git
- `vscode/` - VS Code user settings
- `x11/` - X11 session config (xprofile, xinitrc)
- `scripts/` - Utility automation
- `system/` - System configs (symlinked into `/etc/` via sudo)

### Scripts

- `scripts/push_vault.py` - sync helper for an Obsidian vault repo
- `scripts/sync_repo.py` - bidirectional git sync engine used by macOS LaunchAgents

## Install

```bash
./install.sh        # common (all platforms)
./install.sh --pi   # + Pi 5 configs
./install.sh --mac  # + macOS LaunchAgents for git-synced folders
```

## macOS git sync

`scripts/sync_repo.py` syncs a local git repo bidirectionally against its remote on a schedule. It force-downloads any iCloud-evicted files, commits and pushes local changes, then pulls if behind. On conflict it keeps the local version, saves the remote version as `_conflicted`, and commits the merge resolution automatically. On failure it sends a macOS notification and logs detail to `~/Library/Logs/<name>-sync.log`.

Each synced folder gets its own LaunchAgent plist generated by `install.sh --mac` and loaded into `~/Library/LaunchAgents/`. The script is the shared engine; the plist is the instance config. Folder names are intentionally not committed — they live only in the generated plist.

**To add a new synced folder**, add a plist generation block to the `--mac` section of `install.sh` and re-run `./install.sh --mac`. Copy the existing `repo` block and change:
- `PLIST` filename: `com.brf.<name>-sync.plist`
- `Label`: `com.brf.<name>-sync`
- `REPO_DIR`: path to the local git repo (can be a symlink, e.g. into iCloud)
- `LOGFILE` / `StandardOutPath` / `StandardErrorPath`: use `<name>` consistently

The target directory must already be a git repo with a configured remote and working auth (HTTPS via macOS keychain, or SSH key in keychain via `ssh-add --apple-use-keychain`).

**iCloud folders require Full Disk Access.** If `REPO_DIR` is inside `~/Library/Mobile Documents/` (iCloud Drive), `/usr/bin/python3` needs Full Disk Access: System Settings → Privacy & Security → Full Disk Access → add `/usr/bin/python3`. The script uses `/usr/bin/git` (Xcode shim) which inherits system permissions and does not need separate FDA. Without FDA for python3, git operations fail with "Unable to read current working directory: Operation not permitted".

## Maintenance

`edcloud` bootstrap depends on this repo exposing a working `install.sh` at
`~/src/dotfiles/install.sh`.

If the install entrypoint, its location, or its behavior changes, update
`edcloud` in the same change or immediately after so rebuilds keep working.
