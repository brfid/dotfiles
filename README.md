# dotfiles

Personal system configuration, symlinked into place by `install.sh`.

## Directory layout

- `nvim/` -- Neovim (LazyVim)
- `shell/` -- Zsh, environment variables
- `bash/` -- Bash
- `tmux/` -- Tmux (Pi only)
- `git/` -- Git
- `vscode/` -- VS Code user settings
- `x11/` -- X11 session config (Pi only)
- `scripts/` -- Utility automation
- `system/` -- System configs (symlinked into `/etc/` via sudo)

## Scripts

- `scripts/sync_repo.py` -- Bidirectional git sync engine used by macOS
  LaunchAgents. See [macOS git sync](#macos-git-sync).

## Install

```bash
./install.sh        # common (all platforms)
./install.sh --pi   # + Pi 5 configs
./install.sh --mac  # + macOS LaunchAgents for git-synced folders
```

## macOS git sync

`scripts/sync_repo.py` syncs a local git repo bidirectionally against
its remote on a 15-minute schedule. It:

1. Force-downloads any iCloud-evicted files.
2. Commits and pushes local changes.
3. Pulls remote changes (merge, not rebase).
4. On conflict, keeps the local version and saves the remote version as
   a `_conflicted` sibling file.
5. Pushes the result.

On failure it sends a macOS notification and logs to
`~/Library/Logs/<name>-sync.log`.

### Architecture

Each synced folder gets its own LaunchAgent plist generated by
`install.sh --mac` and loaded into `~/Library/LaunchAgents/`. The
script is the shared engine; the plist is the instance config. Folder
names are intentionally not committed -- they live only in the
generated plist.

### Adding a new synced folder

Add a plist generation block to the `--mac` section of `install.sh`
and re-run `./install.sh --mac`. Copy the existing block and change:

- `PLIST` filename: `com.brf.<name>-sync.plist`
- `Label`: `com.brf.<name>-sync`
- `REPO_DIR`: path to the local git repo (can be a symlink into iCloud)
- `LOGFILE` / `StandardOutPath` / `StandardErrorPath`: use `<name>`
  consistently

### Prerequisites

- The target directory must already be a git repo with a configured
  remote and working auth (HTTPS via macOS keychain, or SSH key in
  keychain via `ssh-add --apple-use-keychain`).

- If `REPO_DIR` is inside `~/Library/Mobile Documents/` (iCloud Drive),
  `/usr/bin/python3` needs Full Disk Access: System Settings -> Privacy
  & Security -> Full Disk Access -> add `/usr/bin/python3`. The script
  uses `/usr/bin/git` (Xcode shim) which inherits system permissions
  and does not need separate FDA. Without FDA for python3, git
  operations fail with "Unable to read current working directory:
  Operation not permitted".

## Maintenance

`edcloud` bootstrap depends on this repo exposing a working
`install.sh` at `~/src/dotfiles/install.sh`.

If the install entrypoint, its location, or its behavior changes,
update `edcloud` in the same change or immediately after so rebuilds
keep working.
